# プロトタイプのセットアップ手順

## まず試す（設定なしでOK）

`index.html` をブラウザで開けば、そのまま動きます。
今は「デモモード」になっていて、データはそのブラウザの中だけに保存されます。流れの確認やUXチェックはこれで十分です。

## 本番用：Googleスプレッドシートと連携する

複数人のデータをちゃんと集めたいときは、以下の手順でスプレッドシートと繋げます。所要時間は15〜20分くらいです。

### 1. スプレッドシートを作る
新しいGoogleスプレッドシートを作成する（名前は何でもOK、例：「無人島の選択_データ」）。

### 2. Apps Scriptを開く
スプレッドシートのメニューから「拡張機能」→「Apps Script」を開く。

### 3. コードを貼り付ける
最初から入っている `myFunction` などのコードを全部消して、`apps-script/Code.gs` の中身を全部コピーして貼り付ける。

貼り付けたら、上部の保存アイコン（フロッピーディスクのマーク）を押す。

### 4. Webアプリとして公開する
右上の「デプロイ」→「新しいデプロイ」を選ぶ。

歯車アイコンから種類を選ぶ画面で「ウェブアプリ」を選択。

以下の設定にする。
- 説明：何でもいい（例：無人島プロトタイプ）
- 次のユーザーとして実行：**自分**
- アクセスできるユーザー：**全員**

「デプロイ」を押す。初回は権限の確認画面が出るので、自分のGoogleアカウントで許可する。

デプロイが終わると、「ウェブアプリ」のURLが表示される。これをコピーする（`https://script.google.com/macros/s/....../exec` の形）。

### 5. HTML側にURLを貼る
`js/config.js` をテキストエディタで開き、以下の部分を探す。

```js
const CONFIG = {
  APPS_SCRIPT_URL: "", // 例: "https://script.google.com/macros/s/xxxxx/exec"
  ...
```

`APPS_SCRIPT_URL: ""` の `""` の中に、コピーしたURLを貼り付けて保存する。

```js
APPS_SCRIPT_URL: "https://script.google.com/macros/s/xxxxx/exec",
```

これで保存すれば、回答が自動でスプレッドシートに送られるようになる。デモモードの警告バナーも消える。

### 6. 動作確認
一度自分で最初から最後まで通しでやってみて、スプレッドシートに以下の5つのシートが自動で作られ、行が追加されていくか確認する。

- **参加者マスタ**：参加者ID・端末情報・開始/終了日時
- **決断ログ**：決断①・②の順位データ、並べ替え回数
- **漂流日記回答**：漂流日記①〜④の回答内容
- **表示データログ**：比較フィードバック画面に表示した平均・N・比較対象者
- **行動ログ**：滞在時間・自由記述の入力時間・ドラッグ履歴・途中離脱

## パイロットデータを先に用意しておく

OC当日に最初の参加者が来たときにも比較対象があるよう、事前にURLの末尾へ `?pilot=1` を付けてアクセスし、何件か自分（や関係者）でパイロットデータを記録しておくのがおすすめです（例: `https://.../index.html?pilot=1`）。パイロットデータは通常の参考データ集計にそのまま使われますが、`isPilot` 列に印が付くので、OC終了後の分析時にはこの列で除外できます。

## 「参加者平均」がまだ少ないとき

`CONFIG.MIN_N_FOR_REAL_AVERAGE`（今は3件、`js/config.js` 内）より決断①のデータがまだ少ないうちは、比較フィードバック画面の「平均順位」は仮の参考値（`js/data.js` の `PRESET_BASELINE`）を表示する。実際のデータが集まってきたら自動で実データに切り替わる。この仮の数値は自由に変更できる。
なお「最も異なる選択をした参加者の例」の方は、参考データが1件でもあればすぐに実データを使う。

## 判断タイプ（功利主義型／中間型／義務論型）の判定について

`js/judgment.js` の `classifyByComposition` という関数で、「食料を渡す2人」の組み合わせだけからその場でリアルタイムに判定している（実利的な役割の2人なら功利主義型、そうでない2人なら義務論型、1人ずつなら中間型）。自由記述の理由に含まれるキーワードは、この判定には使わず、`calcReasonKeywordScore` で別途スコア化して分析用に保存するだけ。キーワードリストは `js/characters.js` の `CHARACTERS` 内の `keywords` で調整できる。

## 質問が出そうなところ

**Q. 複数人が同時にアクセスしても大丈夫？**
A. 大丈夫。それぞれのブラウザで別々にデータが送られる。

**Q. スマホでも動く？**
A. 動きます。ランキングはドラッグでも▲▼ボタンでも並び替えられる。

**Q. データはどこで見れる？**
A. 手順1で作ったスプレッドシートに、シートごとに自動で貯まっていきます。

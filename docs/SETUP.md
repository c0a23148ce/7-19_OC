# プロトタイプのセットアップ手順

## まず試す（設定なしでOK）

`shippai_prototype.html` をブラウザで開けば、そのまま動きます。
今は「デモモード」になっていて、データはそのブラウザの中だけに保存されます。流れの確認やUXチェックはこれで十分です。

## 本番用：Googleスプレッドシートと連携する

複数人のデータをちゃんと集めたいときは、以下の手順でスプレッドシートと繋げます。所要時間は15〜20分くらいです。

### 1. スプレッドシートを作る
新しいGoogleスプレッドシートを作成する（名前は何でもOK、例：「無人島の選択_データ」）。

### 2. Apps Scriptを開く
スプレッドシートのメニューから「拡張機能」→「Apps Script」を開く。

### 3. コードを貼り付ける
最初から入っている `myFunction` などのコードを全部消して、`apps_script_code.txt` の中身を全部コピーして貼り付ける。

貼り付けたら、上部の保存アイコン（フロッピーディスクのマーク）を押す。

### 4. Webアプリとして公開する
右上の「デプロイ」→「新しいデプロイ」を選ぶ。

歯車アイコンから種類を選ぶ画面で「ウェブアプリ」を選択。

以下の設定にする。
- 説明：何でもいい（例：無人島プロトタイプ）
- 次のユーザーとして実行：**自分**
- アクセスできるユーザー：**全員**

「デプロイ」を押す。初回は権限の確認画面が出るので、自分のGoogleアカウントで許可する。

デプロイが終わると、「ウェブアプリ」のURLが表示される。これをコピーする（`https://script.google.com/macros/s/....../exec` の形）。

### 5. HTML側にURLを貼る
`shippai_prototype.html` をテキストエディタで開き、以下の部分を探す。

```js
const CONFIG = {
  APPS_SCRIPT_URL: "", // 例: "https://script.google.com/macros/s/xxxxx/exec"
  ...
```

`APPS_SCRIPT_URL: ""` の `""` の中に、コピーしたURLを貼り付けて保存する。

```js
APPS_SCRIPT_URL: "https://script.google.com/macros/s/xxxxx/exec",
```

これで保存すれば、回答が自動でスプレッドシートに送られるようになる。デモモードの警告バナーも消える。

### 6. 動作確認
一度自分で最初から最後まで通しでやってみて、スプレッドシートに「Step1」「Step2」「Step3」「Step4」というシートが自動で作られ、行が追加されていくか確認する。

## 「参加者平均」がまだ少ないとき

`CONFIG.MIN_N_FOR_REAL_AVERAGE`（今は3人）より参加者が少ないうちは、比較フィードバック画面は仮の参考値（`PRESET_BASELINE`）を表示する。実際のデータが集まってきたら自動で実データに切り替わる。この仮の数値は index.html 内の `PRESET_BASELINE` で自由に変更できる。

## 判断タイプ（A〜D）の判定について

`calcJudgmentType` という関数で、選んだ3人の傾向とQ2の自由記述に含まれるキーワードから自動でスコアをつけている。これはあくまで一次判定用の暫定ルール。キーワードリストは `CHARACTERS` の中の `keywords` で調整できる。

## 質問が出そうなところ

**Q. 複数人が同時にアクセスしても大丈夫？**
A. 大丈夫。それぞれのブラウザで別々にデータが送られる。

**Q. スマホでも動く？**
A. 動きます。ランキングはドラッグでも▲▼ボタンでも並び替えられる。

**Q. データはどこで見れる？**
A. 手順1で作ったスプレッドシートに、シートごとに自動で貯まっていきます。
